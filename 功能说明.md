# 域名工具箱 — 功能说明文档

> 版本：v1.6.1 | 更新日期：2026-02-27
>
> 本文档涵盖除管理后台、账号登录、安全相关以外的所有功能，按模块分类说明。

## 一、端口批量生成（端口生成页）

### 1.1 功能描述

批量为域名随机分配端口，生成完整的 `https://域名:端口` 格式 URL。支持多个端口组，每条域名从选定端口组中随机分配一个端口，避免手动拼接。

### 1.2 端口组说明

| 端口组 | 说明 |
|--------|------|
| **A8** | 常规端口（6001–9663 范围内的 19 个端口） |
| **A-8543** | 特殊端口（3127–9249 范围内的 19 个端口） |
| **Toff** | Toff 专用端口（9003–9979 范围内的 63 个端口） |

### 1.3 使用方式

1. 在文本框中输入域名，每行一个（支持纯域名或完整 URL，粘贴时自动提取 URL）
2. 选择一个或多个端口组（可多选，每条域名从所有选中端口组的端口中随机取一个）
3. 点击「生成端口 URL」，得到带端口的完整 URL 列表
4. 可对每条 URL 单独复制，或点击「复制全部」一次性复制所有 URL
5. 生成后可选择检测工具（ITDOG / 阿里云 / 两者都用），点击「前往检测」直接跳转到域名检测页并自动填入

### 1.4 使用条件

- 无需登录即可使用
- 粘贴时自动过滤重复域名，并提示被过滤的条数
- 手动输入只允许 URL 合法字符（自动过滤中文等非法字符）

## 二、域名检测（域名检测页）

### 2.1 功能描述

对输入的域名进行全国多节点实时速度检测，支持同时使用 ITDOG 和阿里云拨测两个工具，多域名并行后台运行，切换域名时不中断检测。

### 2.2 检测工具

| 工具 | 节点数 | 特点 |
|------|--------|------|
| **ITDOG 测速** | 约 132 个全国节点 | 全国多运营商节点，含海外（香港） |
| **阿里云拨测** | 约 200+ 个全国节点 | 覆盖电信/联通/移动，含 SSL 时间 |

### 2.3 检测结果展示

检测完成后，结果视图包含以下 Tab：

| Tab | 内容 |
|-----|------|
| **全国地图** | 省份着色地图，按平均响应速度分级（绿/黄/橙/红） |
| **运营商统计** | 各运营商（电信/联通/移动）的平均速度、最快/最慢节点 |
| **IP 解析分布** | 各响应 IP 的节点占比（CDN 分布分析） |
| **详细结果** | 所有节点的完整数据表格，含响应 IP、IP 归属地、状态码、总耗时、DNS、连接、下载时间，支持排序和运营商过滤 |

速度分级标准：

| 等级 | 范围 | 颜色 |
|------|------|------|
| 极快 | < 200ms | 绿色 |
| 快 | 200–500ms | 黄绿色 |
| 中等 | 500–1000ms | 橙黄色 |
| 慢 | 1000–2000ms | 橙色 |
| 很慢 | > 2000ms | 红色 |

### 2.4 域名质量评级

每次检测完成后自动计算质量等级（ITDOG 优先，其次阿里云）：

| 等级 | 条件 |
|------|------|
| **优秀** | 失败节点 ≤ 4 且平均延迟 < 5000ms |
| **普通** | 失败节点 ≤ 6 且平均延迟 < 8000ms |
| **极差** | 失败节点 ≤ 8 且平均延迟 < 10000ms |
| **质量差** | 失败节点 > 8 或平均延迟 ≥ 10000ms |

### 2.5 一键复制域名

在左侧域名列表中，hover 域名条目时出现复制按钮（✓ 图标）。

**复制条件（同时满足）：**
1. 该域名已经过「端口生成」工具处理（URL 含非标准端口）
2. 质量评级为「普通」或「优秀」（极差/质量差不可复制）
3. 检测结果未过期（检测完成后 10 分钟内有效）
4. 已登录账号

**复制行为：**
- 将完整 URL（含协议+端口）写入剪贴板
- 每次复制均记录到后端（用于统计）
- 质量不达标时点击会弹出提示说明原因

### 2.6 检测队列与并发

- 系统默认并发 10 个检测任务
- 任务排队时显示当前排队位置、前方任务数、预计等待时间
- 检测中实时显示已接收节点数和进度百分比
- 定时检测执行期间，普通任务最多占用 3 个并发槽（定时任务优先）
- 待检测域名堆积 > 50 时自动升至 30 并发

### 2.7 状态持久化

- 检测状态按当前登录用户 ID 隔离存储（localStorage）
- 刷新页面后自动恢复上次的域名列表、检测结果和选中状态
- 路由切换（跳转到其他页面再回来）时后台检测持续运行，结果不丢失

### 2.8 使用条件

- 输入域名后立即开始后台检测，无需额外点击
- 支持同时检测多个域名（每个域名独立任务）
- 支持从「端口生成」页直接跳转并自动填入域名
- 复制功能需要登录

## 三、定时检测（定时检测页）

### 3.1 功能描述

将域名按分组管理，由调度器定期自动执行检测，无需手动触发。检测结果持久化存储，支持历史记录查询。

### 3.2 分组管理

每个分组包含以下属性：

| 属性 | 说明 |
|------|------|
| **名称** | 分组标识，如「主站域名」「备用线路」 |
| **类别标签** | 用于页面分类展示，相同类别归为一组 |
| **检测工具** | ITDOG 或 阿里云（每个分组独立选择） |
| **检测间隔** | 30分钟 / 1小时（默认）/ 2小时 / 4小时 / 6小时 / 12小时 / 24小时 |
| **备注** | 可选的分组用途说明 |
| **授权状态** | 待授权 / 已授权（只有已授权分组才会被调度器执行） |
| **启用状态** | 启用 / 暂停（暂停后调度器跳过该分组） |

### 3.3 域名管理

- 在分组内添加域名（支持粘贴 URL 批量添加，自动提取 `https://` 链接）
- 每条域名显示：最近检测状态（正常/警告/异常/质量差/待检测）、最近检测时间、检测摘要
- 支持查看每条域名的最近一次检测详情（失败节点分省份汇总）
- 支持查看每条域名的历史检测记录（时间线展示）
- 支持单条复制域名（直接复制到剪贴板）
- 支持批量选择并删除域名

### 3.4 调度器状态

定时检测页顶部显示调度器实时状态：

| 字段 | 说明 |
|------|------|
| **运行状态** | 空闲 / 检测中（显示当前检测任务） |
| **下次检测** | 下次自动执行的时间 |
| **上次检测** | 上次执行的日期时间 |
| **累计检测** | 历史总检测次数 |
| **并发模式** | 正常（10 并发）/ 高并发（30 并发） |
| **活跃/排队** | 当前活跃任务数和排队任务数 |
| **在线人员** | 当前在线的成员列表（站长/管理员/成员分色显示） |
| **暂停原因** | 若定时检测被暂停，显示暂停原因 |

### 3.5 在线人员前置条件

定时检测执行前会检查在线人员状态：

| 在线状态 | 调度器行为 |
|----------|-----------|
| 无任何用户在线 | 暂停本轮检测，等待成员上线 |
| 仅站长在线 | 暂停本轮检测，等待普通成员或管理员上线 |
| 有普通成员或管理员（非站长）在线 | 正常执行检测 |

- 在线判断依据：已登录用户每 30 秒自动上报心跳，5 分钟内有心跳记录视为在线
- 暂停原因会显示在调度器状态卡片中

### 3.6 手动触发检测

- 管理员/站长可点击「立即检测全部」按钮，立即触发所有已授权分组的检测
- 手动触发不受在线人员前置条件限制

### 3.7 质量告警

- 定时检测发现质量差（poor/bad）域名时，自动向所有当时已登录的用户推送弹窗告警
- 告警弹窗显示：分组名称、质量差域名列表、摘要信息（失败节点数/平均延迟）
- 告警弹窗只能手动关闭，不会自动消失
- 多条告警支持堆叠，可逐条查看

### 3.8 权限说明

| 角色 | 可执行操作 |
|------|-----------|
| **普通成员（职员）** | 创建分组、添加域名、查看检测结果 |
| **管理员/站长** | 以上全部 + 授权/撤销分组、手动触发检测、删除任意分组 |

### 3.9 使用条件

- 需要登录才能访问定时检测页
- 新建分组后需管理员授权才会被自动检测
- 分组展开/折叠状态按用户 ID 隔离持久化（刷新后保持）

## 四、站内信通知

### 4.1 功能描述

顶部导航栏右侧的铃铛图标，用于接收系统推送的通知消息（主要来源：定时检测质量告警）。

### 4.2 功能详情

- 有未读消息时铃铛显示红色数字角标（最多显示 99+）
- 点击铃铛弹出通知列表，最多显示最近 50 条
- 每条通知包含：标题、内容、时间，以及关联域名的「一键复制」和「查看分组」快捷按钮
- 「一键复制」：复制告警域名并自动跳转到对应分组
- 「查看分组」：直接跳转到定时检测页的对应分组（带高亮定位）
- 支持「全部已读」操作（一键标记所有消息为已读）
- 支持「删除已读」操作（批量删除所有已读消息）
- 每 30 秒自动轮询一次未读数量

### 4.3 使用条件

- 需要登录才能看到铃铛图标
- 通知只推送给告警发生时已在线（已建立 SSE 连接）的用户

## 五、检测结果详情（失败节点详情弹窗）

### 5.1 功能描述

在定时检测页，点击域名右侧的「详情」图标，查看该域名最近一次检测的失败节点详情。

### 5.2 展示内容

- 检测摘要：总节点数、成功数、失败数、平均延迟
- 质量评级（优秀/普通/极差/质量差）
- 失败节点按省份分组展示（可展开/折叠）
- 慢节点提示（超过 50 秒的节点单独标注）
- 检测工具和检测时间

### 5.3 使用条件

- 需要登录
- 仅显示最近一次检测结果，历史记录通过「历史」按钮查看

---

## 六、历史检测记录

### 6.1 功能描述

在定时检测页，点击域名右侧的「历史」图标，展开该域名的历史检测时间线。

### 6.2 展示内容

- 时间线列表（最近 20 条）
- 每条记录：检测时间、质量评级徽章、检测摘要（失败节点数/平均延迟）
- 点击每条历史记录可查看该次检测的完整节点数据

### 6.3 使用条件

- 需要登录
- 仅显示通过定时检测（调度器）执行的历史记录，手动检测不计入

## 七、功能联动关系

```
端口生成页
  └─ 生成带端口 URL
       └─ 「前往检测」→ 域名检测页（自动填入）
            └─ 检测完成 + 质量达标 → 一键复制域名

定时检测页
  └─ 调度器自动执行
       └─ 发现质量差域名 → 质量告警弹窗（所有在线用户）
                          → 站内信通知（可查看分组/复制域名）
```

---

## 八、注意事项

1. **复制限制**：只有经过端口生成工具处理（含非标准端口）且质量达标（普通/优秀）的域名才可复制，防止误用质量差的域名
2. **检测结果有效期**：域名检测结果 10 分钟后过期，过期后需重新检测才可复制
3. **定时检测前置条件**：无普通成员在线时，定时检测自动暂停，有成员上线后自动恢复
4. **并发限制**：同时最多 10 个检测任务（高负载时自动扩至 30），多余任务排队等待
5. **数据隔离**：域名检测状态和分组展开状态按登录用户 ID 隔离，不同账号互不干扰
